### **Розділ 5. Спостережувані величини та функціонали**

#### 5.1. Оператор Капітальної Ефективності CES(ω)

Центральним спостережуваним у просторі Ω є **Капітальна Ефективність** — міра здатності конфігурації генерувати та утримувати цінність.

**Означення 5.1 (Капітальна Ефективність).** CES: Ω → [0, 100] визначається як:

**CES(ω) = (Fₗ(a) × M_v(v) × Pₜ(t)) - SF(ω)**

де:
- **Fₗ(a)** — ліверидж агента (функція від a ∈ A)
- **M_v(v)** — вектор ринку (функція від v ∈ V)
- **Pₜ(t)** — фазова відповідність (функція від t ∈ T)
- **SF(ω)** — структурне тертя конфігурації

**Теорема 5.1 (Розкладність CES).** За відсутності структурного тертя (SF=0), CES факторизується в добуток незалежних внесків трьох вимірів.

*Доведення.* Безпосередньо з означення: при SF=0 маємо CES(ω) = Fₗ(a) × M_v(v) × Pₜ(t).

**Визначення компонентів:**

**Таблиця 5.1. Значення Fₗ(a) (ліверидж агента)**

| a | Fₗ | Інтерпретація |
|---|-----|---------------|
| ME | 0.7 | Обмежений ліверидж: одна людина може зробити лише обмежено багато |
| WE | 1.0 | Оптимальний ліверидж: команда масштабує зусилля |
| YOU | 0.8 | Ліверидж через клієнтів: залежність від зовнішнього попиту |
| THEY | 0.9 | Мережевий ліверидж: ефект платформи, але з втратою контролю |

**Таблиця 5.2. Значення M_v(v) (вектор ринку)**

| v | M_v | Інтерпретація |
|---|-----|---------------|
| EAST | 1.2 | Інноваційна премія: нові технології дають надприбутки |
| SOUTH | 1.0 | Ринкова норма: дистрибуція дає стабільний дохід |
| WEST | 0.9 | Капіталомісткість: вимагає значних інвестицій |
| NORTH | 0.8 | Регуляторне навантаження: знижує маржинальність |

**Таблиця 5.3. Значення Pₜ(t) (фазова відповідність)**

| t | Pₜ | Інтерпретація |
|---|-----|---------------|
| SPRING | 0.8 | Експериментальна фаза: низькі доходи, високі ризики |
| SUMMER | 1.2 | Фаза зростання: максимальна віддача |
| AUTUMN | 1.0 | Фаза оптимізації: стабільні, але не зростаючі доходи |
| WINTER | 0.6 | Фаза занепаду: стиснення ринку |

**Приклад 5.1 (Розрахунок CES для ідеального стану).** Для ω = (WE, EAST, SUMMER):
- Fₗ(WE) = 1.0
- M_v(EAST) = 1.2
- Pₜ(SUMMER) = 1.2
- SF(ω) ≈ 0.1 (низьке тертя)

CES = (1.0 × 1.2 × 1.2) - 0.1 = 1.44 - 0.1 = 1.34

Нормуючи до шкали 0-100 (множник ≈ 60): 1.34 × 60 = 80.4 ≈ **82**

**Приклад 5.2 (Розрахунок CES для кризового стану).** Для ω = (ME, NORTH, WINTER):
- Fₗ(ME) = 0.7
- M_v(NORTH) = 0.8
- Pₜ(WINTER) = 0.6
- SF(ω) ≈ 0.8 (високе тертя)

CES = (0.7 × 0.8 × 0.6) - 0.8 = 0.336 - 0.8 = -0.464

Нормуючи: від'ємні значення прирівнюємо до 0. Отже, **CES = 0**

#### 5.2. Проекційні оператори: ризик, монетизація, ліверидж

Кожен спостережуваний параметр бізнесу може бути отриманий як **проекція** стану ω на певний підпростір або як згортка з відповідним функціоналом.

**Означення 5.2 (Оператор проекції).** Для підмножини координат I ⊂ {1,2,3,4,5,6}, проекція π_I: Ω → {0,1}^|I| виділяє відповідні біти стану.

**Приклад 5.3 (Проекція на агента).** π_{1,2}(ω) дає код агента (2 біти), що визначає тип лівериджу.

**Означення 5.3 (Профіль ризику).** Функціонал ризику R: Ω → [0,1] визначається як:

R(ω) = (1 - Pₜ(t)) × (1 - M_v(v)) × (1 + SF(ω)) / 3

**Інтерпретація.** Ризик високий, коли:
- Фаза несприятлива (Pₜ малий)
- Вектор ринку слабкий (M_v малий)
- Структурне тертя високе (SF великий)

**Таблиця 5.4. Рівні ризику**

| R(ω) | Рівень ризику | Приклади станів |
|------|---------------|------------------|
| 0.0-0.2 | Низький | (WE, EAST, SUMMER), (THEY, SOUTH, AUTUMN) |
| 0.2-0.4 | Помірний | (ME, SOUTH, SPRING), (WE, WEST, AUTUMN) |
| 0.4-0.6 | Підвищений | (YOU, NORTH, SUMMER), (THEY, EAST, WINTER) |
| 0.6-0.8 | Високий | (ME, NORTH, AUTUMN), (YOU, WEST, WINTER) |
| 0.8-1.0 | Критичний | (ME, NORTH, WINTER), (YOU, EAST, WINTER) |

**Означення 5.4 (Модель монетизації).** Функціонал монетизації M: Ω → {Subscription, Transaction, Asset, Service} визначається як:

M(ω) = f(π_A(ω), π_V(ω))

де f задається таблицею:

**Таблиця 5.5. Мапа монетизації (Хто × Куди → Модель)**

| A\V | EAST | SOUTH | WEST | NORTH |
|-----|------|-------|------|-------|
| **ME** | Ліцензування | Консалтинг | Інвестування | Лобіювання |
| **WE** | R&D контракти | Дистрибуція | Private Equity | Нормотворчість |
| **YOU** | Платформа SaaS | Маркетплейс | Краудфандинг | Страхування |
| **THEY** | API доступ | Реклама | Біржі | Держзакупівлі |

#### 5.3. Спектр станів: класифікація конфігурацій

На основі значень CES та SF можна класифікувати всі 64 стани за їхньою структурною стійкістю та потенціалом.

**Теорема 5.2 (Спектральна класифікація).** Простір Ω розбивається на 4 класи стійкості:

1. **Атрактори** (Stable) — стани з високою CES та низьким SF
   - CES > 70, SF < 0.3
   - Бізнеси в цих станах мають тенденцію залишатися в них
   - Приклади: (WE, EAST, SUMMER), (THEY, SOUTH, AUTUMN)

2. **Метастабільні** (Metastable) — висока CES, але значне SF
   - CES > 70, 0.3 ≤ SF < 0.6
   - Потребують постійних зусиль для підтримання
   - Приклади: (ME, EAST, SUMMER), (YOU, SOUTH, AUTUMN)

3. **Транзитні** (Transient) — середня CES, через які проходять траєкторії
   - 40 ≤ CES ≤ 70
   - Бізнеси рідко зупиняються тут надовго
   - Приклади: (WE, WEST, SPRING), (THEY, NORTH, SUMMER)

4. **Репелери** (Unstable) — низька CES, високе SF
   - CES < 40, SF > 0.6
   - Стани, яких уникають або швидко покидають
   - Приклади: (ME, NORTH, WINTER), (YOU, EAST, WINTER)

**Спектральна діаграма 5.1.** (тут буде візуалізація розподілу 64 станів за CES та SF)

**Теорема 5.3 (Про зв'язність спектру).** Множина атракторів утворює зв'язний підграф у просторі Ω (від будь-якого атрактора можна досягти будь-якого іншого, рухаючись лише через атрактори).

*Доведення.* Аналіз всіх 64 станів показує, що атрактори (близько 12-16 станів) розташовані так, що між ними завжди є шлях довжиною 1-2 через інші атрактори. Наприклад, (WE, EAST, SUMMER) сусідить з (WE, SOUTH, SUMMER), який сусідить з (THEY, SOUTH, SUMMER) і т.д.

### **Розділ 6. Спостереження та вимірювання в просторі Ω**

#### 6.1. Методологія ідентифікації стану ω для реального бізнесу

Як визначити, в якому стані ω перебуває реальна компанія? Це ключове питання прикладного застосування теорії.

**Процедура 6.1 (Ідентифікація стану).**

**Крок 1. Визначення агента (WHO)**
Питання для діагностики:
- Хто приймає ключові рішення? (ME — одна особа, WE — команда)
- Хто є бенефіціаром цінності? (YOU — клієнт, THEY — система/платформа)
- Як розподілена відповідальність?

Критерії:
- ME: соло-засновник, сімейний бізнес, самозайнятий
- WE: партнерство, ТОВ з кількома співвласниками, стартап зі співзасновниками
- YOU: бізнес, побудований навколо конкретного клієнта, кастомні рішення
- THEY: платформа, маркетплейс, мережева структура, корпорація

**Крок 2. Визначення вектора (WHERE)**
Питання:
- Де накопичується основна цінність? (EAST — інтелектуальна власність, технології)
- Що є ключовим активом? (SOUTH — клієнтська база, канали дистрибуції)
- Куди спрямовані інвестиції? (WEST — фінансові активи, WEST — регуляторні переваги)

Критерії:
- EAST: патенти, R&D, технологічна перевага, ноу-хау
- SOUTH: дистрибуційна мережа, клієнтська база, маркетинг
- WEST: капітал, інвестиції, власність, активи
- NORTH: ліцензії, дозволи, регуляторний статус, стандарти

**Крок 3. Визначення фази (WHEN)**
Питання:
- Яка динаміка ринку? (SPRING — зародження, експерименти)
- Як змінюється виручка? (SUMMER — швидке зростання)
- Яка конкурентна ситуація? (AUTUMN — консолідація, оптимізація)
- Які настрої в індустрії? (WINTER — спад, трансформація)

Критерії:
- SPRING: експерименти, відсутність стабільної моделі, high risk
- SUMMER: зростання 20%+ річних, розширення, найм
- AUTUMN: стабілізація, оптимізація, фокус на ефективність
- WINTER: падіння, консолідація, пошук нового

**Приклад 6.1 (Ідентифікація Tesla, 2020).**
- WHO: WE (команда Маска + інженери) з елементами THEY (платформа)
- WHERE: EAST (технології) + WEST (капітал)
- WHEN: SUMMER (масштабування виробництва)

Підсумок: домінантний стан (WE, EAST, SUMMER) з високою CES.

#### 6.2. Шкали та нормування: від якісних оцінок до кількісних

Для практичного застосування необхідно перетворити якісні спостереження в кількісні показники.

**Процедура 6.2 (Кількісна оцінка SF).**

SF(ω) = w₁·SF_self + w₂·SF_pairwise, де w₁ + w₂ = 1

**SF_self** — внутрішня неузгодженість кожного виміру:
- SF_self = (s_A + s_V + s_T) / 3
- s_X = 0, якщо X узгоджений сам із собою, 1 — якщо ні

Наприклад, для (ME, NORTH, WINTER):
- ME у WINTER: s_A = 0.8 (індивід у фазі занепаду — дуже важко)
- NORTH у WINTER: s_V = 0.7 (регуляції в занепаді — втрата сенсу)
- WINTER сам по собі: s_T = 0.3 (зима є зимою)

**SF_pairwise** — попарна неузгодженість:
- SF_pairwise = (s_AV + s_VT + s_TA) / 3
- s_XY — міра того, наскільки добре X поєднується з Y

**Таблиця 6.1. Матриця попарної узгодженості A×V (фрагмент)**

| A\V | EAST | SOUTH | WEST | NORTH |
|-----|------|-------|------|-------|
| ME | 0.1 | 0.3 | 0.4 | 0.7 |
| WE | 0.0 | 0.1 | 0.2 | 0.4 |
| YOU | 0.3 | 0.1 | 0.3 | 0.5 |
| THEY | 0.4 | 0.2 | 0.1 | 0.2 |

**Приклад 6.2 (Розрахунок SF для (ME, NORTH, WINTER)).**
- SF_self = (0.8 + 0.7 + 0.3)/3 = 0.6
- s_AV = 0.7 (ME-NORTH)
- s_VT: NORTH-WINTER = 0.6
- s_TA: WINTER-ME = 0.8
- SF_pairwise = (0.7 + 0.6 + 0.8)/3 = 0.7
- SF = 0.5·0.6 + 0.5·0.7 = 0.65

#### 6.3. Похибки вимірювань та невизначеність спостерігача

Важливе методологічне зауваження: **стан ω не є об'єктивною властивістю бізнесу**. Це результат спостереження, який залежить від позиції спостерігача, часу спостереження та обраної шкали.

**Теорема 6.1 (Про невизначеність спостерігача).** Для будь-якого реального бізнесу існує інтервал невизначеності Δω ⊂ Ω, такий що:
1. Різні спостерігачі можуть локалізувати бізнес у різних точках Δω
2. Один спостерігач у різні моменти може бачити різні точки Δω
3. Δω має діаметр не менше 1 (тобто як мінімум сусідні стани нерозрізненні)

**Приклад 6.3 (Невизначеність для стартапу).** Молодий технологічний стартап може бути одночасно:
- (ME, EAST, SPRING) — якщо дивитися на засновника
- (WE, EAST, SPRING) — якщо дивитися на команду
- (YOU, EAST, SPRING) — якщо дивитися на перших клієнтів

Δω = { (ME,EAST,SPRING), (WE,EAST,SPRING), (YOU,EAST,SPRING) }, діаметр = 2.

**Наслідок 6.2.** При практичному застосуванні слід працювати не з точковими оцінками, а з "хмарами" станів та ймовірнісними розподілами.
